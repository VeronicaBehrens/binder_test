<hr /> <h2 id="1.-R-Syntax-Basics">1. R Syntax Basics¶</h2>
<h3 id="Assignment">Assignment¶</h3><p>In R, we assign values to variables using <- (the preferred R style) or =. Both work, but <- is the R convention.</p> 
gene_name <- "POU5F1"    # character (text)
fold_change <- 2.5       # numeric
is_significant <- TRUE   # logical (TRUE/FALSE)
<h3 id="Comments">Comments¶</h3><p>Anything after # is a comment and is ignored by R:</p> 
# This is a comment
gene_count <- 500  # This is also a comment
<h3 id="Function-Calls">Function Calls¶</h3><p>R uses functions extensively. Functions take arguments inside parentheses:</p> 
print("Hello, stem cells!")
sqrt(16)       # square root
length(c(1,2,3))  # length of a vector
<h3 id="Getting-Help">Getting Help¶</h3>
  • ?function_name opens help for a function
  • ??search_term searches all help pages
  • help("function_name") is equivalent to ?

<hr /> <h2 id="2.-Data-Types-and-Structures">2. Data Types and Structures¶</h2>
<h3 id="Basic-Data-Types">Basic Data Types¶</h3><p>R has several core data types:</p> <thead> <th>Type</th> <th>Example</th> <th>Description</th> </thead> <tbody> </tbody> <tbody></tbody>
numeric 2.5, 100 Numbers (decimals or integers)
character "POU5F1" Text strings
logical TRUE, FALSE Boolean values
factor factor(c("ESC","MPS")) Categorical data with defined levels
<h3 id="Vectors">Vectors¶</h3><p>A vector is the most common data structure in R — a sequence of elements of the same type. Create vectors with c() (combine):</p> 
gene_names <- c("POU5F1", "NANOG", "SOX2", "KLF4")  # character vector
fold_changes <- c(1.5, 2.3, -0.8, 0.5)              # numeric vector
<h3 id="Data-Frames">Data Frames¶</h3><p>A data frame is a table (like a spreadsheet) where each column can be a different type. This is the most common structure for bioinformatics data. We'll load one from CSV shortly.</p> <h3 id="Inspecting-Objects">Inspecting Objects¶</h3>
  • str() — structure of an object (most useful!)
  • class() — the type/class of an object
  • summary() — statistical summary
  • head() — first few rows
  • dim() — dimensions (rows × columns)
  • names() or colnames() — column names

<hr /> <h2 id="4.-Indexing-and-Subsetting">4. Indexing and Subsetting¶</h2>
<h3 id="Data-Frame-Indexing">Data Frame Indexing¶</h3><p>Access data in data frames using several methods:</p> <thead> <th>Syntax</th> <th>What it does</th> </thead> <tbody> </tbody> <tbody></tbody>
df[i, j] Row i, column j
df[i, ] Row i (all columns)
df[, j] Column j (all rows)
df$col A single column by name
df[, "col"] Same as above, by name
df[df$col > x, ] Rows where col > x (logical indexing)
<h3 id="Logical-Indexing">Logical Indexing¶</h3><p>This is one of the most powerful patterns in R — filter rows based on a condition:</p> 
# Get cells with high total counts
high_count_cells <- cell_meta[cell_meta$total_counts > 2000000, ]
